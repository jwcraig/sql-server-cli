## Directory Purposes

**src/cli/**
- Purpose: CLI argument parsing and structure definitions
- Contains: `args.rs` (all arg structs and parsing), `mod.rs` (module export)
- Key files: `args.rs` - 600+ lines of clap-based argument definitions
- Subdirectories: None

**src/commands/**
- Purpose: Subcommand implementations
- Contains: One file per command (20+ commands)
- Key files: `mod.rs` (dispatcher), `sql.rs`, `describe.rs`, `tables.rs`, `compare.rs`
- Pattern: Each command has a `run(args, cmd_args) -> Result<()>` function

**src/config/**
- Purpose: Configuration loading and merging
- Contains: Schema definitions, environment parsing, config file loading
- Key files: `loader.rs` (main loading logic), `env.rs` (env var parsing), `schema.rs` (types)
- Subdirectories: None

**src/db/**
- Purpose: SQL Server database operations
- Contains: Connection management, query execution, type conversions
- Key files: `client.rs` (connection), `executor.rs` (query running), `queries.rs` (SQL builders)
- Subdirectories: None

**src/output/**
- Purpose: Result formatting and display
- Contains: Format-specific renderers
- Key files: `table.rs` (comfy-table), `json.rs`, `csv.rs`, `mod.rs`
- Subdirectories: None

**src/safety/**
- Purpose: Read-only mode enforcement
- Contains: SQL statement analysis
- Key files: `read_only.rs` (SQL blocking logic), `mod.rs`
- Subdirectories: None

**tests/**
- Purpose: Integration tests for CLI
- Contains: Test files using assert_cmd
- Key files: `help_test.rs`, `init_command_test.rs`, `integration_commands_test.rs`
- Subdirectories: `common/` (shared test utilities)

**assets/**
- Purpose: Static files installed by integration commands
- Contains: Skill files for agent integration
- Key files: `GEMINI.md` (Gemini CLI extension)
- Subdirectories: None

## Key File Locations

**Entry Points:**
- `src/main.rs` - CLI entry, error handling, logging init
- `src/lib.rs` - Library entry, module exports

**Configuration:**
- `Cargo.toml` - Project metadata, dependencies
- `config.example.yaml` - Example configuration file
- `.githooks/pre-push` - Pre-push validation hook

**Core Logic:**
- `src/commands/mod.rs` - Command dispatcher
- `src/cli/args.rs` - All CLI argument definitions
- `src/db/client.rs` - Database connection logic
- `src/config/loader.rs` - Configuration merging

**Testing:**
- `tests/*.rs` - Integration tests
- `tests/common/mod.rs` - Shared test utilities

**Documentation:**
- `README.md` - User-facing documentation
- `AGENTS.md` - Coding guidelines for AI agents

## Naming Conventions

**Files:**
- snake_case.rs for all Rust modules
- Descriptive names matching functionality (e.g., `foreign_keys.rs`, `query_stats.rs`)
- `mod.rs` for module definitions

**Directories:**
- snake_case for all directories
- Plural for collections: commands/, tests/

**Special Patterns:**
- `*_test.rs` for test files in tests/ directory
- No `_test.rs` suffix in src/ (Rust uses `#[cfg(test)]` inline)

## Where to Add New Code

**New Command:**
- Implementation: `src/commands/{command_name}.rs`
- Args: Add struct to `src/cli/args.rs`, add variant to `CommandKind`
- Dispatch: Add match arm in `src/commands/mod.rs`
- Tests: `tests/{command_name}_test.rs`

**New Config Option:**
- Schema: `src/config/schema.rs`
- Loading: `src/config/loader.rs`
- Env var: `src/config/env.rs`

**New Output Format:**
- Implementation: `src/output/{format}.rs`
- Export: Add to `src/output/mod.rs`

**Utilities:**
- Shared helpers: Add to appropriate module or create new in src/
- Database queries: `src/db/queries.rs`

## Special Directories

**target/**
- Purpose: Build artifacts
- Source: Generated by cargo
- Committed: No (in .gitignore)

**.planning/**
- Purpose: Project planning and codebase documentation
- Source: Manual and generated
- Committed: Yes

**.private/**
- Purpose: Private/local files (docs backup, etc.)
- Source: Local development
- Committed: No (in .gitignore)
